HOW TO INSTALL FOG PXE SERVER
==============================

SOURCE 1: FOG Project Official Documentation (docs.fogproject.org)
-------------------------------------------------------------------
Before rushing into installing FOG you want to make sure you check the requirements.
The installation instructions assume that you have a freshly installed server available that only contains the minimal set of packages.

Updating fog is essentially the same process: instead of creating a fresh clone of the repo, you do a git pull and run the installer again.

The preferred method of getting FOG is via Git.

INSTALLATION STEPS:
1. Install git:
   - For Debian/Ubuntu: sudo apt-get -y install git
   - For RedHat/CentOS: sudo dnf -y install git

2. Clone the FOG repository:
   sudo -i
   cd /root
   git clone https://github.com/FOGProject/fogproject.git
   cd fogproject

3. Choose your branch:
   - dev-branch 'dev': stable option with testing
   - stable branch: longer-term stable version
   - working-1.6: beta version for testing new features

   To switch to dev-branch:
   cd /root/fogproject
   git fetch --all
   git checkout dev-branch

   To switch back to stable:
   git fetch --all
   git checkout stable

4. Run the installer (must be done as root):
   sudo -i
   cd /root/fogproject/bin
   ./installfog.sh

   The installer has various switches for running silently. See Fog installer command line options for details.

5. Pre-installation questions:
   - SELinux: Disable if enabled (FOG will give problems when SELinux is enabled)
   - Local Firewall: Disable if you don't know how to set up rules
   - OS Selection: Confirm the distribution the installer guesses
   - Installation mode: Choose N for normal FOG server (master node)
   - Default Network interface: Confirm or change the network interface for PXE booting

SOURCE 2: LinuxShout Guide (linux.how2shout.com)
------------------------------------------------
What Is FOG Project and How Does It Work?
FOG (Free Open-source Ghost) is a network-based computer cloning and management solution.
Unlike traditional imaging tools that require physical media, FOG uses PXE (Preboot Execution Environment) to boot client computers directly from the network.

Basic workflow:
- Capture: Set up a reference machine, FOG captures this as an image stored on your server
- Deploy: Client computers boot from the network via PXE, FOG pushes the stored image
- Manage: FOG provides inventory tracking, Active Directory integration, printer management, and scheduled tasks

SYSTEM REQUIREMENTS:
Hardware:
- CPU: Any modern 64-bit processor (multi-core recommended)
- RAM: Minimum 2 GB, recommended 4 GB or more
- Storage: Depends on images. A single Windows 11 image can range from 15-50 GB. Plan for at least 100 GB for the FOG partition.

Network:
- Gigabit Ethernet strongly recommended
- Wired network connection (Wi-Fi won't work for PXE boot)
- Either control over your DHCP server or ability to run FOG's built-in DHCP
- Network switches that support PXE/TFTP traffic

SUPPORTED OPERATING SYSTEMS:
- Ubuntu: 20.04 LTS, 22.04 LTS, 24.04 LTS or 26.04 LTS
- Debian: 11 (Bullseye), 12 (Bookworm)

THINGS TO PREPARE BEFORE INSTALLING:
1. Assign a Static IP Address - Client machines won't find your PXE server if the IP changes
2. Use a Wired Connection - Wi-Fi won't work for PXE boot
3. Understand Secure Boot Implications - May need to disable Secure Boot in client BIOS/UEFI

STEP-BY-STEP INSTALLATION ON UBUNTU OR DEBIAN:

Step 1 - Update the System:
   sudo apt update && sudo apt upgrade -y
   If kernel was updated, reboot: sudo reboot

Step 2 - Install Required Packages:
   (Install git as shown above)

Step 3 - Download FOG Project from GitHub:
   sudo -i
   cd /root
   git clone https://github.com/FOGProject/fogproject.git
   cd fogproject

Step 4 - Run the FOG Installation Script:
   sudo -i
   cd /root/fogproject/bin
   ./installfog.sh

Step 5 - Complete FOG Web Interface Setup:
   Follow the installer prompts and configure the web interface after installation.

DEFAULT LOGIN CREDENTIALS:
   Check the FOG documentation or installer output for default credentials.

COMMON INSTALLATION PROBLEMS:
- DHCP Conflicts: Ensure FOG has control over DHCP or configure properly
- Firewall Blocking Services: Disable firewall or configure rules for FOG services
- PXE Boot Not Working: Verify network interface, TFTP server, and DHCP settings

SECURITY CONSIDERATIONS:
- FOG itself is lightweight but image storage demands space
- Consider SELinux implications
- Configure firewall rules for FOG services
- Review community support and security updates

FOG VS. PAID TOOLS:
   FOG is free, open-source software that can replace expensive proprietary solutions.

FINAL THOUGHTS:
   FOG is suitable for educational institutions, corporate IT departments, MSPs, and repair shops.
   Community support is available through the FOG project GitHub repository and documentation.
