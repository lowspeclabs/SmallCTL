What Went Right

1. Task Classification Fix Worked

Problem solved: Coding challenges were previously misclassified as `remote_execute` because the full prompt (including SSH instructions) was fed to the classifier. This caused the model to try SSH for local coding tasks, hitting host-key mismatches and getting blocked by remote-task guards.

Fix applied:

  • Updated task_classifier.py:task_is_local_coding_target() to detect strong coding markers ("build a self-contained python script", "unittest", "./temp/*.py") and return local_execute regardless of SSH mentions elsewhere in the prompt.
  • Split sysadmin/coding prompt templates in run_baseline.py so coding prompts no longer contain SSH credential instructions.

Result: All 20 coding challenges now correctly classify as `local_execute`. Log confirms:

Task mode: local_execute
Intent: requested_shell_exec
Tags: shell_exec phase_execute python

2. Step Budget Increase Helped Sysadmin

Fix applied: Hard step-budget nudge increased from 12 → 40 steps in `lifecycle_nodes.py:614`.

Result: Sysadmin challenges 11, 12, and 16 completed successfully at ~585s each. Previously these would have been killed at step 12. More hard sysadmin tasks now have room to complete.

3. SSH Guard for Local Tasks (tool_dispatch.py)

Fix applied: Hard block on SSH tools when `task_mode == "local_execute"`.

Result: No more coding tasks attempting `ssh_exec` and hitting host-key errors.

4. Escalation Guidance Respected (dispatcher.py)

Fix applied: If the bigger model's escalation guidance recommends `shell_exec`, the remote-task guard no longer blocks it.

Result: No guard loops where escalation says "use shell_exec" but the guard says "no, use ssh_exec."

What Went Wrong

Critical Issue: LM Studio Context Window Too Small (4096 tokens)

Impact: All 20 coding challenges failed immediately (~8s each).

Error from backend: 

The number of tokens to keep from the initial prompt is greater than 
the context length (n_keep: 10128 >= n_ctx: 4096). Try to load the 
model with a larger context length, or provide a shorter input.

Why this happens:

  • The model (qwen3.5-4b-mtp) supports 60k context, but LM Studio loaded it with only 4096 tokens.
  • The harness system prompt alone is 5,991 tokens (tool schemas, instructions, recovery guidance).
  • Coding task prompts total ~9,500 tokens.
  • The prompt exceeds the backend's n_ctx before any completion is generated.

Why sysadmin was less affected:

  • Sysadmin tasks stream tool outputs (SSH command results) which are short and external. The prompt doesn't embed large code blocks.
  • Coding tasks must embed the full Python script + unittest cases in the prompt, making them much larger.

Verdict: This is a **server-side configuration issue**, not a harness or script bug.

Sysadmin Hard Challenges Still Time Out (8/20)

Pattern: Challenges 03, 04, 09, 10, 14, 15, 16, 18 all hit the 600s timeout.

Common symptoms from logs:

  • context_lane_dropped events strip observations from the prompt
  • task_complete_verifier_loop_status_autocontinue — model calls task_complete but verifier fails, triggering repair loops
  • write_recovery_attempt / write_recovery_declined — file writes are attempted but rejected
  • Step counts reach 30-40 steps before timeout

Root cause: Hard sysadmin tasks (e.g., cross-domain incident triage, configuration drift audit) require:

  1. Multiple SSH discovery commands
  2. File reads across multiple remote paths
  3. Correlation of disparate data sources
  4. Report writing and verification

The 4B model is simply too slow at this complexity level. TTTF (time to first token) is 12-18s, and each step takes 20-40s. A 30-step task takes 10-15 minutes, exceeding the 600s timeout.

One Sysadmin Challenge Failed (challenge-20)

Status: `failed` at 499.83s (not timeout).

Likely cause: Task reached step budget (40) without completing, or hit a guard that forced failure.

Conclusion

The prompt separation and classification fixes are working correctly. Coding tasks are now properly recognized as local and no longer attempt SSH. However, the entire coding category is blocked by a server-side context limit that prevents any prompt from fitting into LM Studio's 4096-token window.

Next step: Reconfigure LM Studio with a 32k+ context window and rerun. Expected outcome: coding challenges should start completing (at least easy/medium), and sysadmin scores should remain similar or improve slightly with the higher step budget.
